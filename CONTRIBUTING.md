# Contributing to AI Industrial Design Assistant

Thank you for your interest in contributing to AI Industrial Design Assistant.

This project is an open-source web application for industrial designers, product design students, and creators who want to generate structured design project documents from a simple product idea or design competition topic.

## How You Can Contribute

You can contribute in several ways:

- Improve the user interface

- Improve the placeholder design outputs

- Add new design document sections

- Add OpenAI API integration

- Improve documentation

- Add example design projects

- Fix bugs

- Suggest new features for industrial design workflows

## Project Setup

1. Clone the repository.

```bash

git clone [https://github.com/jihyeBaek0/ai-industrial-design-assistant.git](https://github.com/jihyeBaek0/ai-industrial-design-assistant.git)

cd ai-industrial-design-assistant

```

2. Create a virtual environment.

```bash

python -m venv .venv

```

3. Activate the virtual environment.

For Windows PowerShell:

```bash

.\.venv\Scripts\Activate.ps1

```

4. Install dependencies.

```bash

python -m pip install -r requirements.txt

```

5. Run the app.

```bash

python [app.py](http://app.py)

```

Then open:

```text

[http://127.0.0.1:5000/](http://127.0.0.1:5000/)

```

## Development Guidelines

Please try to keep the project:

- Beginner-friendly

- Easy to run locally

- Useful for real industrial design workflows

- Clear and well-documented

- Simple before adding complex features

## Suggested Contribution Areas

### Frontend

- Improve responsive layout

- Add animations

- Improve accessibility

- Improve visual hierarchy

- Add dark mode

### Backend

- Improve placeholder generation logic

- Add input validation

- Add better error handling

- Prepare OpenAI API integration

### Design Workflow

- Add design competition mode

- Add CMF strategy templates

- Add user research templates

- Add design critique prompts

- Add portfolio case study generator

## Pull Request Guidelines

Before opening a pull request:

1. Make sure the app runs locally.

2. Keep changes focused and easy to review.

3. Update the README if your change affects setup or usage.

4. Add examples when possible.

5. Use clear commit messages.

Example commit messages:

```text

Improve responsive card layout

Add sample design project output

Update README with setup instructions

Add OpenAI API integration placeholder

```

## Code Style

This project currently uses:

- Python for the Flask backend

- HTML for templates

- CSS for styling

- JavaScript for frontend interaction

Please keep the code readable and beginner-friendly.
